# 项目模型与功能积分定价表

> 说明：本文档将原始“任务类型 / 消耗积分 / 内容举例”整理为更适合 AI 检索、解析和决策的 Markdown 结构。  
> 单位统一为“积分”。缺失或不明确的价格使用 `待确认` 标记。

## 1. 计费字段约定

| 字段 | 含义 |
|---|---|
| `任务类型` | 用户在产品中触发的功能或模型 |
| `节点/入口` | 对应的节点、入口或使用场景 |
| `模型/通道` | 具体模型名、普通/尊享通道、Fast/Pro/Ultra 档位 |
| `分辨率/规格` | 1K、2K、4K、720P、1080P 等规格 |
| `消耗积分` | 单次任务或指定单位下的积分消耗 |
| `计费方式` | 单次、单张、4 张、每秒、公式、免费、待确认等 |
| `备注` | 适用场景、特殊规则或原始说明 |

### 1.1 斜杠价格解释

| 价格格式 | 在本文档中的解释 |
|---|---|
| `20/80`，出现在图片生成主表 | `单张图片 / Generate 4 四张图` |
| `80/170`，出现在高清放大 | `普通通道 / 尊享通道` |
| `80/100`，出现在 KlingO3 | `无声视频每秒 / 有声视频每秒` |
| `80/120`，出现在 Wan2.7 | `720P / 1080P`，原始文档未明确是否按秒计费 |
| `10+20*张数` | 固定分析费 10 + 每张生成费 20 |
| `10+40` | 分析费 10 + 生成费 40，合计 50 |

- 上传图片

- 拆分分镜脚本提示词

- 拼接图片

- AI 提取调色板

| AI 视频反推提示词 | 30 |

### 13.3 图片生成价格选择逻辑

```text
如果是普通图片生成：
  根据 通道类型 + 档位 + 分辨率 选择单张积分。
  如果使用 Generate 4，则使用四张图积分。

如果是画布编辑、扩图、局部编辑、多图融合、视觉变换：
  使用“画布图片扩图/编辑”价格表。
  不使用 Generate 4 价格。
```

### 13.4 视频价格选择逻辑

```text
如果是 Seedance 2.0 / Seedance 2.0 Fast：
  按 分辨率 × 视频时长秒数 计费。
  480P Fast = 80.6 积分/秒
  720P Fast = 96.6 积分/秒
  480P 标准 = 100 积分/秒
  720P 标准 = 120 积分/秒
  1080P 标准 = 300 积分/秒

如果是 KlingO3：
  无声 = 80 积分/秒
  有声 = 100 积分/秒

如果是 Wan2.7：
  720P = 80
  1080P = 120
  但原始文档未明确单位，调用前应确认是否按秒计费。
```
